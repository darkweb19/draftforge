# P01-T02 — Atomic transitions, event log, and config discovery

Status: ready

## Objective

Complete the Phase 1 exit gate: a project can be resumed and handed between harnesses
without manual file edits, and every accepted change is recorded.

## Owned paths

- `src/state/`
- `src/config/`
- `src/commands/`
- `test/fixtures/`
- Relevant tests and documentation.

## Dependencies

- P01-T01

## Acceptance criteria

- State transitions are validated against the task state machine in `docs/PROTOCOL.md` and rejected when illegal.
- Each accepted transition appends a redacted event to `.draftforge/runs/<run-id>/events.jsonl`; the log is append-only.
- Config discovery loads `.draftforge/config.json`, applies `.draftforge/config.local.json` overrides, validates against the schema, and reports actionable errors.
- `status` and `doctor` report project health, including drift between state and `SESSION.md`.
- Fixtures exist for fresh, resumed, and corrupted projects, and tests use them.
- No provider, CLI login, or API key is required.

## Exclusions

- No model calls, provider adapters, or planning logic.
