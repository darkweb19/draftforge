# P03-T02 — Harness adapters: Codex CLI and Claude Code

Status: backlog

## Objective

Implement the two subscription-backed adapters that satisfy the model-runner
port by invoking the locally authenticated `codex` and `claude` commands as
child processes, parsing their output into a `ModelResponse`, and reusing the
P03-T01 reliability and redaction. Both must pass the shared adapter
contract-test suite against a faked process boundary.

## Acceptance criteria

- The Codex CLI and Claude Code adapters implement the port by invoking the
  local `codex` and `claude` commands through one shared process transport,
  passing the prompt without embedding or logging credentials.
- Both adapters pass the P03-T01 shared contract-test suite against a faked
  process boundary; tests never spawn a real CLI.
- Non-zero exit, missing command, and timeout map to typed, redacted errors, and
  transient failures are distinguished from terminal ones for retry.
- `provider-default` lets the harness choose its model; an explicit model string
  is forwarded as the command's model selection.
- Capability discovery reports transport `harness` and auth mode `local-cli`
  without probing the network.
- `npm run check` passes.

## Verification

Run `npm run check`. When `codex` or `claude` is authenticated locally, run the
guarded live smoke test for that adapter; it is skipped when the command is
absent.

- No `doctor` changes and no `plan` command wiring.
