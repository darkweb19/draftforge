# P03-T03 — API adapters: OpenAI and Anthropic

Status: backlog

## Objective

Implement the two API-backed adapters that satisfy the model-runner port over
HTTP using environment keys, reusing the P03-T01 reliability and redaction, and
passing the shared adapter contract-test suite against an injected `fetch`. No
vendor SDK is added; requests use the runtime `fetch`.

## Owned paths

- `src/providers/api/http.ts` (shared HTTP transport and auth-header injection)
- `src/providers/api/openai-api.ts`
- `src/providers/api/anthropic-api.ts`
- `src/providers/registry.ts` (register the two API adapters; replaces the
  placeholders created in P03-T01)
- `test/providers/api/*.test.ts` with an injected `fetch`
- `README.md`
- Canonical project state and handoff files for this task.

## Acceptance criteria

- The OpenAI and Anthropic adapters implement the port over HTTP with the
  runtime `fetch`, reading `OPENAI_API_KEY` and `ANTHROPIC_API_KEY` only from the
  environment and never logging or persisting them.
- Both adapters pass the P03-T01 shared contract-test suite against an injected
  `fetch`; tests make no real network call.
- HTTP 401/403 map to terminal authentication errors; 429 and 5xx map to
  transient, retryable errors; the request carries the routed role, model, and
  reasoning level.
- `provider-default` resolves to each provider's documented default model without
  hardcoding a volatile id in configuration or domain.
- Redaction guarantees the key never appears in any error, log, or evidence.
- No vendor SDK dependency is added to `package.json`.
- `npm run check` passes.

## Verification

Run `npm run check`. When an API key is present, run the guarded live smoke test
for that provider; it is skipped when the key is unset.
