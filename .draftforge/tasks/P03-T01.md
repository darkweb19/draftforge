# P03-T01 — Model-runner factory, role routing, and adapter reliability

Status: ready

## Objective

Establish the Phase 3 provider foundation behind the existing `ModelRunner`
port: a factory that builds a role-routed runner from `ProjectConfig`, a shared
adapter contract with capability discovery, and reusable reliability (timeout,
bounded retry, and secret redaction). No real provider transport ships in this
task; the concrete adapters arrive in P03-T02 and P03-T03.

## Acceptance criteria

- `createModelRunner(config)` returns a `ModelRunner` that routes each
  `ModelRole` to its configured adapter, model, and reasoning level, resolving
  `provider-default` without embedding a volatile model id.
- The adapter contract exposes capability discovery (id, transport, supported
  roles, auth mode) as pure data, performing no network or process call.
- A reusable contract-test suite asserts request/response shape, role
  passthrough, redaction, timeout, and retry behavior for any adapter.
- Reliability enforces a per-call timeout, retries only transient failures a
  bounded number of times, and never retries authentication or contract errors.
- Redaction removes secret material (API keys, `Authorization` values, and
  configured environment values) from every text, log, and error the layer
  surfaces; a test proves no secret can leak.
- `src/domain/` and `src/application/` still depend only on the port; provider
  code imports no vendor SDK.
- `npm run check` passes.
